# Projeto CI/CD  
![CI](https://img.shields.io/github/actions/workflow/status/AndersonDominguesRS/Assesment-Pipelines/deploy.yml?branch=main)


O Git é uma ferramenta essencial para manter o controle das versões do código e permitir que o processo de integração contínua funcione corretamente. Ele ajuda a organizar as mudanças feitas no projeto e garante que cada modificação possa ser acompanhada e revisada quando necessário. No contexto de DevOps, o Git acaba servindo como ponto central onde o código é atualizado e onde os pipelines são acionados automaticamente sempre que algo novo é enviado para o repositório.

As branches permitem que o desenvolvimento aconteça sem interferir diretamente na versão principal do sistema. Isso evita problemas e facilita o trabalho em equipe, já que cada pessoa pode trabalhar em uma parte diferente sem causar conflitos imediatos. As tags e releases também são importantes porque marcam versões específicas do software, ajudando a identificar o que foi entregue em cada etapa e facilitando o processo de deploy.

No ciclo DevOps, o Git acaba funcionando como uma base que conecta desenvolvimento e operações, permitindo automação, testes e entregas mais rápidas. Ele garante que o código esteja sempre versionado e que as mudanças passem por validações antes de chegar em produção, deixando o processo mais seguro e organizado.
